## GitHub運用について
ブランチの切り方はGitFlowを取り入れてみる。

コミットメッセージの書き方
- fix：バグ修正
- add：新規（ファイル）機能追加
- update：機能修正（バグではない）
- remove：削除（ファイル）
- clean：整理（リファクタリング等）
